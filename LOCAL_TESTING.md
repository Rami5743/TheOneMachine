# בדיקה מקומית אמינה

מקומית כל הקבצים נטענים מהדיסק באופן מיידי, ולכן **לא רואים** את האיטיות
שקיימת בטעינה ראשונה דרך האינטרנט — וגם לא את סמל הטעינה ואת יתרון הטעינה-מראש.
הדף הזה מסביר איך לשחזר את התנאים האמיתיים מקומית.

## 1. להריץ דרך שרת HTTP (לא לפתוח את הקובץ ישירות)

פתיחת `index.html` ישירות (כתובת `file://`) לא משחזרת נכון את התנהגות הטעינה
והמטמון. הריצו שרת קטן מתוך תיקיית הפרויקט:

```bash
# פייתון (מותקן ברוב מחשבי Windows כ-py או python)
python -m http.server 8000
```

ואז גשו אל <http://localhost:8000> בדפדפן.

## 2. לשחזר טעינה ראשונה איטית

בדפדפן (Chrome/Edge):

1. פתחו את כלי הפיתוח: `F12`.
2. עברו ללשונית **Network**.
3. סמנו **Disable cache** (רק כשה-DevTools פתוח).
4. בתפריט ה-throttling בחרו **Slow 4G** (או צרו פרופיל מותאם איטי יותר).
5. רעננו (`Ctrl+R`).

עכשיו הטעינה תתנהג כמו בפעם הראשונה ברשת: תראו את שעון החול המסתובב
במרכז המסך, ותראו שהשקף מתגלה רק כשהכול (וקטור + רסטר) מוכן.

## 3. דגל בדיקה מהיר לסמל הטעינה — `?slideDelay`

בלי צורך ב-throttling, אפשר להוסיף לכתובת פרמטר שמאלץ כל שקף להתגלות
לאחר השהיה מינימלית (במילישניות):

```
http://localhost:8000/index.html?slideDelay=2000
```

כל שקף יציג את שעון החול לפחות 2 שניות ואז יתגלה. בלי הפרמטר — אין שום
השפעה, כך שזה לא נוגע בחוויה האמיתית. (מקסימום 20000ms.)

## 4. אזהרת מטמון בעריכת שקפים

זו הסיבה שראית עיכוב **רק כשמשנים שקפים**: כתובות קבצי ה-`app.js` וה-CSS
נושאות מספר גרסה (`?v=...`), ולכן שינוי קוד תמיד נטען מחדש אוטומטית. אבל
קבצי התמונה של השקפים (`assets/panels/*.png`, `*.svg`) **אינם** נושאים
מספר גרסה. אם עורכים תמונת שקף קיימת (אותו שם קובץ), הדפדפן עלול להציג את
הגרסה השמורה במטמון.

הפתרון בזמן בדיקה: **רענון קשיח** — `Ctrl+Shift+R` — מושך את כל הקבצים
מחדש, כולל התמונות.
